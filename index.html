Image Resizing

QIS stores images in a server native filesystem - the limits on storage are thus the size of the physical storage medium - but you need only store a single instance of the image. Pre-sizing of images is no longer a regular activity due to QIS's ability to resize images on demand.

When an image is requested QIS will query its cache of previously served images for a suitable base image before reading the original from file if needed. A customized library is called to render the image at the new size before saving it to cache and returning the new image.

QIS can pre-emptively generate subsequent versions of the image in the background if configured to do so so that new iterations of the image are already in cache ready for a fast response, frequently in only a few milliseconds. Such optimization results in QIS being one of the fast image resizing engines available.

For more information, please see https://github.com/quru/qis.

Watermarking & Protection

QIS can overlay any image on top of any other image. You can watermark an image before it leaves the server regardless of its size or orientation. Whilst the standard version will support jpg, png, tif and gif formatted images as watermarks, the premium addition will also support svg images which can therefore act as a mask, producing final images of varying shapes.

The overlay system currently requires the overlaid image to be stored in the QIS server repository. Having selected the image the following attributes can be change:

  • Size - by percentage
  • Position - by compass point, eg top left, centre bottom etc
  • Opacity - by percentage with 10% being barely visible and 100% solid

Variations and combinations of these attributes, together with clever use of transparency in the overlaid image can produce almost any effect needed.

Image Analytics

By default, every image served and every download provided is recorded with the volume of the data served and a time stamp. This provides a very clear view of the load on the server as a whole or the demand for any particular image. A zoomable graph showing the performance is provided in the main analytics page or on each images’ details.

If statistics are not wanted on an image by image basis or on a template by template basis then the addition of “&stats=0” in the image URL will immediately disable them. This is particularly useful of you want to only record renderings of certain sizes.

QIS can be easily integrated into an CRM or other customer centric system by embedding a unique reference into an “&xfref=<value>” tag in any image url. If QIS has been correctly configured, this value will be sent to any api to record exactly when that image was viewed with that reference. This can be used to prove that the image was opened and thus ensure statistics are as accurate as possible.

Bonhams is one of the four major Georgian auction houses selling 200,000 fine painting, ceramics, jewellery, motor cars and many other categories from its main sale rooms in London, New York and Hong Kong. It has a major on-line presence where customers will inspect images of lots before deciding to bid on them. But Bonhams also depends on high quality glossy printed catalogues which are printed through a fully colour-managed process. QIS holds all of Bonhams images in a 15TB store and provides every thing from thumbnails to print ready PDFs.

Katie Walker Furniture

Katie Walker designs and sells high quality, award winning furniture; her work can be found in major galleries and collections as well as on the High Street. Managing photographic records of the development, manufacture and finished pieces is a major task that has to be constantly kept to. Using QIS enables Katie to keep images grouped in directories yet shared with press and customers in cross group portfolios. Being able to quickly recover historical images from the server when discussing modifications or changes with manufacturers is as important as being able to find just the right shot to fill the gap in the latest lifestyle magazine.

UK component distributor

A major component distributor, that also sells direct, found that their wholesale customers would simply scrape their website of component images and pricing, place the images on their own sites and then reduce the price by a couple of percent to undercut their supplier and major competitor. QIS came to the record by enabling them to embed both branding and the current price in the images in such a way that their retailers could not copy and undercut. At a stroke, QIS gained them a significant percentage of profit.

Extra Features

Imaging operations can be combined together and saved as templates. QIS supports the Python-Pillow library and ImageMagick package for its core image processing engine. Available operations include:

  • Rotation
  • Resizing
  • Cropping and tiling
  • Vertical and horizontal flip
  • Image format conversion
  • Stripping of image metadata to reduce file size
  • Enhancements in the Premium Edition include:
  • Sharpen and blur
  • Overlays and watermarking
  • Image colorspace conversion
  • PDF conversion to and from images
  • Colour profile conversion and removal for print publishing
  • File types
  • The Standard Edition supports file types: gif, jpg, png, tif. In the Premium edition, supported file types vary depending on your version of ImageMagick, but the following are enabled by default:
  • General image formats: bmp, dcm, gif, jpg, png, ppm, psd, svg, tga, tif, xcf
